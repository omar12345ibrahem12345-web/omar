Machine Learning: The Engine of Modern AI

Traditional programming: Rules + Data = Answers. Machine Learning: Data + Answers = Rules.

Instead of humans writing explicit if-then rules, ML algorithms discover patterns in data automatically. This paradigm shift is why AI can now handle tasks—like recognizing faces or translating languages—that resisted rule-based approaches for decades.

Three Learning Paradigms

Supervised Learning

Labeled data → Learn mapping from inputs to known outputs. Classification (spam/not spam) and regression (house prices). Most common paradigm.

Unsupervised Learning

No labels → Discover hidden structure. Clustering (customer segments), dimensionality reduction (visualization), anomaly detection (fraud).

Reinforcement Learning

Rewards & penalties → Learn by interacting with an environment. Games (AlphaGo), robotics, trading, recommendation systems.

Key Insight: The Loss Function

Everything in ML reduces to minimizing a mathematical function that measures "how wrong" the model is. The choice of loss function encodes what we care about—accuracy, precision, calibrated probabilities, fairness.

# Simplified conceptual example
def mean_squared_error(y_true, y_pred):
    return ((y_true - y_pred) ** 2).mean()

def cross_entropy(y_true, y_pred):
    return -(y_true * log(y_pred) + (1-y_true) * log(1-y_pred)).mean()

Neural Networks & Deep Learning

When neural networks have many layers ("deep"), they can learn hierarchical representations—edges → shapes → objects → concepts—automatically from raw data.

Why Depth Matters

Layer 1: Edges

Detect simple patterns: lines, curves, color contrasts. Equivalent to V1 visual cortex.

Layer 2-3: Textures

Combine edges into textures, corners, simple shapes. Fur, fabric, metal surfaces.

Layer 4-6: Parts

Object components: eyes, wheels, handles, letters. Compositional understanding emerges.

Layer 7+: Objects

Whole objects, scenes, concepts. "Golden retriever," "sunset beach," "code editor."

Inside a Neural Network

The mathematical machinery—accessible without calculus.

Each artificial neuron computes: output = activation(Σ weightᵢ × inputᵢ + bias)

  • Weights determine how much each input matters
  • Bias shifts the activation threshold
  • Activation function (ReLU, GELU, sigmoid) introduces non-linearity—without it, deep networks would collapse to linear models

ReLU (max(0, x)) is the workhorse: simple, sparse gradients, mitigates vanishing gradient problem.

  • Dense/Linear: Every neuron connects to every neuron in previous layer. Good for tabular data, final classification heads.
  • Convolutional (CNN): Shared weights scan local regions. Translation invariance—perfect for images.
  • Attention (Transformer): Dynamic weighting of all positions. Context-aware representations—powers LLMs.
  • Recurrent/RNN: Sequential processing with memory. Largely superseded by Transformers.

Training = Forward pass + Loss computation + Backward pass (backprop) + Weight update.

  1. Forward: Input flows through network → prediction
  2. Loss: Compare prediction to target (e.g., cross-entropy)
  3. Backward: Chain rule computes gradient of loss w.r.t. every weight
  4. Update: Optimizer (SGD, Adam) adjusts weights opposite to gradient

Modern frameworks (PyTorch, TensorFlow, JAX) automate backprop via automatic differentiation.

Embeddings map discrete items (words, images, users) to continuous vectors where semantic similarity = geometric proximity.

  • Word2Vec/GloVe: Static word vectors (king - man + woman ≈ queen)
  • BERT/Transformer embeddings: Contextual—same word, different vector based on context
  • Image embeddings: CLIP aligns text and image in same space
  • Used for: search, recommendations, clustering, transfer learning, zero-shot classification
Interactive embedding explorer →
  • Epoch: One full pass through the training dataset
  • Batch: Subset processed at once (GPU memory constraint). Batch size affects gradient noise & generalization.
  • Learning Rate: Step size for weight updates. Too high → divergence. Too low → slow convergence. Often scheduled (warmup + decay).
  • Regularization: Dropout, weight decay, data augmentation, early stopping—prevent overfitting.
  • Scaling Laws: Performance predictably improves with more data, parameters, compute (Chinchilla, Kaplan et al.).

Large Language Models: How ChatGPT Works

LLMs are Transformers trained on massive text corpora with a simple objective: predict the next token.

The Transformer Architecture

Tokenization

Text → integers (tokens). Subword tokenization (BPE) balances vocabulary size vs. sequence length. ~100k tokens typical.

Embeddings + Position

Token IDs → dense vectors. Positional encoding adds order information (Transformers have no inherent sequence awareness).

Self-Attention

Each token attends to all others: Q×Kᵀ scores → softmax → weighted sum of V. Parallelizable, long-range dependencies.

Feed-Forward + Residual

Per-token MLP (expand → activate → project). Residual connections + LayerNorm enable very deep stacks (96+ layers).

Two-Stage Training

1

Pre-training (Self-Supervised)

Next-token prediction on trillion-token corpora (web, books, code). Learns grammar, facts, reasoning patterns, world knowledge. Expensive: $10M–$100M+ compute.

2

Fine-tuning / Alignment

Instruction tuning (follow commands) + RLHF/RLAIF (human preferences). Makes model helpful, harmless, honest. Cheaper: $10K–$1M.

How AI Generates Output

Inference-time mechanics—what happens when you prompt an LLM.

Autoregressive Generation

Models generate one token at a time, feeding each new token back as input. Cannot "plan ahead" or backtrack—fundamental constraint.

Details →

Sampling Strategies

Greedy: Always highest probability. Temperature: Controls randomness (0 = deterministic, 1 = distribution). Top-k / Top-p: Truncate tail of distribution.

Context Window

Fixed maximum tokens (4K–2M+). Includes prompt + generated output. Longer context = more memory/compute. Retrieval-augmented generation (RAG) extends effective knowledge.

Hallucination Mechanism

Models maximize likelihood, not truth. Plausible-sounding but false outputs arise from: training data errors, distribution mismatch, lack of verification mechanism.
