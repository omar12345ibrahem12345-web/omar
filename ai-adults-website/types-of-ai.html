Types of Artificial Intelligence

From narrow specialists to hypothetical superintelligence—understand the spectrum of AI capabilities and where today's tools actually sit.

Explore the Spectrum → How They Work

The Capability Spectrum

AI isn't one thing—it's a spectrum from highly specialized tools to hypothetical general intelligence.

Narrow AI (ANI)

Artificial Narrow Intelligence — Systems designed for specific tasks. All production AI today falls here.

  • Chess/Go engines (AlphaZero, Stockfish)
  • Recommendation systems (Netflix, TikTok, Amazon)
  • Speech recognition (Siri, Alexa, Whisper)
  • Computer vision (face ID, medical imaging)
  • Spam filters, fraud detection, translation
Key trait: Superhuman at trained task, zero transfer to untrained tasks.
See applications →

Generative AI

Creates new content rather than just classifying or predicting. The breakthrough category of 2022-present.

  • Text: GPT-4, Claude, Llama, Gemini
  • Images: DALL-E 3, Midjourney, Stable Diffusion
  • Code: GitHub Copilot, Cursor, Codeium
  • Audio: ElevenLabs, Suno, MusicLM
  • Video: Sora, Runway Gen-3, Luma Dream Machine
Key trait: Learns distribution of training data to sample novel instances.
How it works →

AGI (Artificial General Intelligence)

Human-level across all cognitive tasks — learn, reason, plan, transfer knowledge. Does not exist yet.

  • No consensus on definition or timeline
  • Estimates range: 2027–2050+ (experts disagree wildly)
  • Key test: Can it do any intellectual task a human can?
  • Requires: continual learning, causal reasoning, world models
  • Benchmarks: ARC-AGI, GAIA, MMLU (current models score 80-90%)
Key trait: Flexible generalization—not just pattern matching on training distribution.
AGI timeline debate →

ASI (Artificial Superintelligence)

Vastly exceeds human intelligence in all domains. Purely speculative—philosophical concept.

  • Recursive self-improvement → intelligence explosion
  • Could solve: aging, energy, physics, space travel
  • Risks: alignment problem, control, misuse
  • Key thinkers: Bostrom, Yudkowsky, Russell, Hinton
  • Not imminent—requires AGI breakthrough first
Key trait: Capabilities beyond human comprehension or control.

By Learning Paradigm

How the system acquires its capabilities—this determines data needs, use cases, and limitations.

Paradigm Data Required How It Learns Typical Use Cases Limitations Examples
Supervised Labeled (input → output pairs) Minimize error on known answers Classification, regression, prediction Expensive labeling; limited to seen patterns Spam filter, medical diagnosis, price prediction
Unsupervised Unlabeled (raw data only) Find structure, clusters, compression Anomaly detection, segmentation, dimensionality reduction No ground truth; hard to evaluate Customer segments, fraud detection, topic modeling
Self-Supervised Unlabeled (creates own labels) Predict masked/missing parts of input LLMs, representation learning, pretraining Massive compute; proxy task ≠ downstream task BERT, GPT, CLIP, MAE, SimCLR
Reinforcement Learning Environment interaction (rewards) Maximize cumulative reward via trial/error Games, robotics, trading, recommendation Sample inefficient; reward design hard; unstable AlphaGo, RLHF, robotics control, Plasmo
Transfer Learning Pre-trained + labeled target data Adapt general features to specific task Fine-tuning LLMs, few-shot adaptation Catastrophic forgetting; domain mismatch LoRA, adapters, prompt tuning, instruction tuning

Foundation Models

Large models (LLMs, vision transformers) pre-trained via self-supervision, then adapted. The dominant paradigm since 2018.

Major Model Families You Should Know

The landscape moves fast—these are the key lineages shaping AI today.

  • GPT-4 / GPT-4o: Flagship multimodal models. Strong reasoning, coding, 128K context.
  • GPT-4o-mini: Cost-efficient small model. 15¢/1M input, 60¢/1M output.
  • o1 / o1-mini: Reasoning models with "chain of thought" for complex math/science.
  • API access: Chat Completions, Assistants, Batch, Realtime APIs.
  • Key trait: Closed-source, best-in-class for general tasks, strong ecosystem.
API docs & pricing →
  • Claude 3.5 Sonnet: Best-in-class coding, 200K context, artifacts for code/visualization.
  • Claude 3 Opus: Highest capability (legacy), 200K context.
  • Claude 3 Haiku: Fast, cheap, 200K context.
  • Constitutional AI: Alignment via principles, not just RLHF. Less refusal, more nuance.
  • Key trait: Strong on long-context analysis, coding, honest about uncertainty.
Claude API guide →
  • Llama 3.1 405B: Flagship, rivals GPT-4o. 128K context. Requires ~800GB VRAM (BF16).
  • Llama 3.1 70B/8B: Practical for self-hosting. 70B ~40GB VRAM (quantized).
  • Llama 3.2: Multimodal (11B, 90B) + lightweight edge models (1B, 3B).
  • License: Llama 3 Community License—commercial use OK with restrictions (>700M users need license).
  • Ecosystem: Ollama, vLLM, llama.cpp, Hugging Face, Groq, Together AI.

  • Gemini 1.5 Pro: 2M token context (largest production). Native multimodal.
  • Gemini 1.5 Flash: Fast, cheap, 1M context. Great for high-volume.
  • Gemini Nano: On-device (Pixel phones). Privacy-first.
  • Integration: Deep Workspace (Gmail, Docs, Sheets), Vertex AI platform.
  • Key trait: Massive context window, strong video/audio understanding.
Vertex AI guide →
  • Mistral: Mixtral 8x7B/8x22B (MoE), Mistral Large, Codestral. Apache 2.0 / commercial.
  • Qwen (Alibaba): Qwen2.5 (0.5B–72B), Qwen2-VL, Qwen-Coder. Strong multilingual, Apache 2.0.
  • Nemotron (NVIDIA): Ultra-efficient distillation. Nemotron 3 Ultra beats GPT-4o on some benchmarks.
  • Phi (Microsoft): Phi-3/3.5 small language models (3.8B–14B). Remarkable quality/size ratio.
  • Command R/R+ (Cohere): Optimized for RAG, tool use, enterprise. 128K context.

Choosing the Right AI for Your Problem

Decision framework: match your task to the appropriate AI approach.

Quick Decision Guide

Your Task Recommended Approach Example Tools Data Needed
"Classify this document / image / transaction" Supervised classification (fine-tuned BERT, ViT, XGBoost) Hugging Face, Roboflow, Azure AI Labeled examples (100s–1000s)
"Predict next quarter's sales / equipment failure" Time series forecasting / tabular regression Chronos, Prophet, XGBoost, DataRobot Historical records with timestamps
"Generate marketing copy / code / images" Generative AI (LLM, diffusion, code model) ChatGPT, Claude, Copilot, Midjourney Prompt + optional few-shot examples
"Find similar items / cluster customers" Embeddings + vector search / clustering Pinecone, Weaviate, Qdrant, FAISS Raw data (text, images, behavior logs)
"Answer questions from our internal docs" RAG (Retrieval-Augmented Generation) LangChain, LlamaIndex, Azure AI Search Document corpus (PDFs, wikis, tickets)
"Automate a multi-step workflow" Agentic AI (tool use, planning, memory) LangGraph, AutoGen, CrewAI, OpenAI Assistants API access, function definitions, eval sets
"Detect anomalies / fraud / defects" Unsupervised / self-supervised anomaly detection Isolation Forest, DeepSAD, Reconstruction models Mostly normal data, few/no labels
"Control a robot / optimize a process" Reinforcement Learning / Control theory Ray RLlib, Stable Baselines3, MuJoCo Simulator or safe real-world environment
